Documento de Requisitos

SuperSystem

1.Introdução

1.1 Propósito do Documento de Requisitos

Esse documento especifica as características e funcionalidades do sistema

SuperSystem. Tem como público alvo os clientes e a equipe de desenvolvimento.

1.2 Escopo do Produto

O Sistema SuperSystem visa ser usado por lojistas e caixas do setor de SuperMercado para controle de estoque, venda e finanças. Ele tem como intuito auxiliar de forma inteligente o controle geral da loja de forma simples e fácil.

2.Descrição Geral

2.1 Perspectiva do Produto

O SuperSystem será feito como Api/Desktop (ainda em decisão) utilizando a linguagem Java, utilizando Banco de Dados MySQL

2.2 Funções do Produto

Controle de vendas, estoque, finanças e RH.

2.3 Características do Usuário

Administradores do sistema, atendentes de caixas, gerentes e gestores.

3.Requisitos

3.1 Requisitos Funcionais

Requisitos técnicos do software que compõe o sistema, que descrevem ações que o sistema deve estar apto a executar, ou seja, o que o sistema deve fazer.

3.1.1 Requisitos Essenciais

RF_01: Sistema de Login
                Para entrada dependendo do nível de privilégio do usuário.

RF_02: Registro de Produto
                Funções de registro de produto, deletar ou adicionar produto ao banco.

RF_03: Controle de Produto
                Funções de registro do produto em estoque, adicionar ou retirar os produtos do estoque.


RF_04: Interface de caixa
                Criação de tela de controle de produtos para atendentes em caixa.

3.1.2 Requisitos Importantes
RF_05: Controle de finanças
                Controle geral de finanças do total de corrente no dia de trabalho.

RF_06: Interface coerente

                Criação de estilo de design para ser seguidos pelas telas.

3.1.3 Requisitos Desejáveis

RF_07: Cadastro de Cliente
                Cadastro de clientes associando-os à compra.

3.2 Requisitos Não-Funcionais

Requisitos técnicos do software que compõe o sistema, que descrevem atributos que o sistema deve possuir ou restrições sob as quais ele deve operar

3.2.1 Requisitos Essenciais

RFN_01:Interface Intuitiva
        Para uso de funcionarios, com feedback de botões.

3.2.2 Requisitos Importantes

RFN_02:Visualização de listagem de produtos
        Lista os produtos disponíveis no estabelecimento.

3.2.3 Requisitos Desejáveis

RFN_03: Login em duas etapas
